## 1. Install the Supabase CLI

```bash
# macOS
brew install supabase/tap/supabase

# Windows (scoop)
scoop bucket add supabase https://github.com/supabase/scoop-bucket.git
scoop install supabase

# Linux / npm
npm i -g supabase
```

Verify:
```bash
supabase --version
```

---

## 2. Log in to Supabase

```bash
supabase login
```
Opens a browser to authenticate with your Supabase account. You must have access to the project (workspace owner can invite you).

---

## 3. Initialize Supabase in your local clone

From the project root:
```bash
supabase init
```
This creates a `supabase/` directory if one isn't present. This repo already contains `supabase/config.toml` and `supabase/migrations/` — **do not overwrite them**. If prompted, decline overwrites.

---

## 4. Link to this Cloud project

```bash
supabase link --project-ref knkwiiywfkbqznbxwqfh
```
You'll be asked for the **database password**. Get it from:
- Lovable → Cloud → Database → Connection settings, **or**
- the workspace owner

## 5. Required environment variables

Create a `.env.local` (gitignored) for local scripts. **Do not edit `.env`** — Lovable manages that file.

```bash
# Public (safe to expose in client)
VITE_SUPABASE_URL=https://knkwiiywfkbqznbxwqfh.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=sb_publishable_bMlmBZJQFG-J7oh5Q65oBw_-ALsyLwg
VITE_SUPABASE_PROJECT_ID=knkwiiywfkbqznbxwqfh

# Server-only (NEVER commit, NEVER expose to the client)
SUPABASE_URL=https://knkwiiywfkbqznbxwqfh.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<get from Cloud → Settings → API>
SUPABASE_DB_URL=postgresql://postgres:<DB_PASSWORD>@db.knkwiiywfkbqznbxwqfh.supabase.co:5432/postgres
```

Where to find them:
- **Service role key** → Lovable → Cloud → Settings → API keys
- **DB password / DB URL** → Lovable → Cloud → Database → Connection settings

## 6. Common workflows

### Pull remote schema
```bash
supabase db pull
```

### Push a new migration
```bash
supabase migration new add_my_table
# edit the generated SQL file
supabase db push
```
> Prefer creating migrations via Lovable (chat) — they're applied automatically and stay in sync.

### Generate TypeScript types
The app already has auto-generated types at `src/integrations/supabase/types.ts` (managed by Lovable — do not edit). If you need a local copy:
```bash
supabase gen types typescript --project-id knkwiiywfkbqznbxwqfh > types.generated.ts
```

### Run the seed verifier
```bash
SUPABASE_URL=https://knkwiiywfkbqznbxwqfh.supabase.co \
SUPABASE_SERVICE_ROLE_KEY=<your-key> \
USER_ID=<your-auth-uid> \
npx tsx scripts/verify-seed.ts
```

### Test edge functions locally
```bash
supabase functions serve <function-name> --env-file .env.local
```

## 7. Troubleshooting

| Symptom | Fix |
|---|---|
| `Project not found` on link | Wrong ref. Use `knkwiiywfkbqznbxwqfh`. |
| `permission denied` on `db push` | You're not a member of the Supabase project. Ask the workspace owner to invite you. |
| Types out of sync | Pull latest from git — Lovable regenerates `src/integrations/supabase/types.ts` on every schema change. |
| `.env` keeps changing | That's expected. Lovable owns `.env`. Use `.env.local` for your own vars. |
