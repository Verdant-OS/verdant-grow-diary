# Pheno Tracker Pro — production release runbook

Operator guide for the three-part release gate: **local-only preflight →
deployed-build fingerprint → live role smoke**, followed by the automated
release receipt. The receipt itself
([pheno-tracker-pro-release-receipt.md](./pheno-tracker-pro-release-receipt.md))
is GENERATED by `scripts/releases/write-pheno-release-receipt.mjs` — do not
hand-edit it; record evidence in the input artifacts instead.

The gate targets `https://verdantgrowdiary.com` only, uses dedicated
production test accounts and existing production-safe fixture hunts, never
seeds production, and never uses `service_role`.

## Sequence

```bash
# 1. Local-only preflight — no network. Prints variable NAMES only.
#    0 READY / 1 FAIL (bad confirm or conflicting E2E_BASE_URL) / 2 BLOCKED (missing inputs)
bun run test:pheno-live-smoke:preflight

# 2. Deployed-build fingerprint — records bundle id/SHA-256/headers to
#    artifacts/.../deployed-build.json. Set PHENO_EXPECTED_LIVE_BUILD_ID to the
#    identifier you just published (exact bundle id/filename, or >=8-hex-char
#    SHA-256 prefix); a mismatch exits 1.
PHENO_EXPECTED_LIVE_BUILD_ID=<published-bundle-id> bun run release:pheno:build-id

# 3. Full live runner — preflight → reachability → fingerprint → sessions →
#    Playwright (chromium-authed, real sessions, no mocking). Writes
#    live-smoke-summary.json including the 12-checkpoint matrix.
bun run test:pheno-live-smoke

# 4. Record the schema spot-check and manual checks (below), then write the receipt.
#    GO exits 0; anything less is HOLD (exit 2).
bun run release:pheno:receipt

# To refresh the receipt BEFORE all evidence exists (always HOLD, never GO):
bun run release:pheno:receipt:partial
```

Required local inputs (set only in the current shell; never commit, never
paste values into chat/PRs/logs):

```
E2E_PHENO_LIVE_SMOKE_CONFIRM=RUN_LIVE_PHENO_SMOKE
E2E_PHENO_{FREE,PRO,FOUNDER,CANCELED}_{EMAIL,PASSWORD}
E2E_PHENO_HUNT_ID_MISSING_EVIDENCE
E2E_PHENO_HUNT_ID_COMPARISON_READY
```

## Production schema spot-check

Run in the production Supabase SQL editor, then record the results in
`artifacts/release-readiness/pheno-tracker-live-smoke/schema-spot-check.json`:

```sql
select column_name
from information_schema.columns
where table_schema = 'public'
  and table_name = 'pheno_hunts'
  and column_name in ('evidence_goals', 'notes', 'setup_completed_at')
order by column_name;

select proname
from pg_proc
where proname = 'has_pheno_tracker_entitlement';

select tablename, policyname, permissive, cmd
from pg_policies
where schemaname = 'public'
  and tablename like 'pheno_%'
  and policyname ilike '%pro%'
order by tablename, policyname;
```

```jsonc
// schema-spot-check.json
{
  "columns": ["evidence_goals", "notes", "setup_completed_at"],
  "entitlementFunctionCount": 1,
  "restrictivePolicyTableCount": 13,
  "allProPoliciesRestrictive": true,
  "ownerSelectVerified": true
}
```

The receipt marks the schema check PASS only when all three columns are
present, exactly one entitlement function exists, all 13 `pheno_*` write
tables have RESTRICTIVE Pro policies, and owner SELECT behavior is verified.

## Manual release checks

Record in
`artifacts/release-readiness/pheno-tracker-live-smoke/manual-release-checks.json`.
Checkpoints 6 (hunt setup persistence) and 9 (missing-evidence navigation)
have no automated proof in the live smoke and stay PENDING until manual
evidence is recorded here:

```jsonc
// manual-release-checks.json
{
  "expectedBuildId": "index-XXXXXXXX",          // the identifier you published
  "publishedAt": "YYYY-MM-DDTHH:MM:SSZ",
  "operator": "name",
  "decisionOwner": "name",
  "deployment": {
    "noWhiteScreen": "PASS",
    "evidence": "manual browser check, timestamp",
    "consoleErrors": "PASS",
    "consoleEvidence": "DevTools clean, timestamp"
  },
  "billing": { "required": false, "status": "NOT_REQUIRED", "evidence": "no billing change in this release" },
  "rollback": {
    "priorVersionIdentified": "PASS",
    "migrationPosture": {
      "status": "PASS",
      "classification": "NON_ADDITIVE_WITH_ROLLBACK_PLAN",
      "summary": "Owner-only Pheno hardening included a documented policy removal.",
      "exceptions": [
        {
          "migration": "20260709180000_pheno_hunts_owner_only_and_stress_scale_index.sql",
          "changeType": "DROP_POLICY",
          "scope": "public.pheno_hunts",
          "description": "Removed the operator SELECT and UPDATE policies (\"Operators view all pheno_hunts\", \"Operators update all pheno_hunts\").",
          "impact": "Operator cross-tenant access removed; owner SELECT access unchanged.",
          "rollbackProcedure": "Recreate the two operator policies from repository history only if operator access must be restored."
        }
      ]
    },

```

Manual checkpoint entries override smoke-derived statuses (both directions —
a manual FAIL beats an automated PASS).

## GO requires ALL of

1. Production URL and main bundle reachable (fingerprint artifact PASS).
2. Expected build identifier matches the observed bundle — reachability
   alone never proves release identity.
3. No white screen; no unexpected console errors (manual, recorded).
4. Schema spot-check PASS (3 columns / 1 function / 13 RESTRICTIVE tables /
   owner SELECT).
5. Live smoke PASS with **zero failed and zero skipped** tests.
6. All 12 checkpoints PASS (automated proof or recorded manual evidence).
7. Billing disposition resolved.
8. Rollback readiness complete: prior version identified, structured
   migration rollback posture (`ADDITIVE` with no exceptions, or
   `NON_ADDITIVE_WITH_ROLLBACK_PLAN` with a complete rollback procedure for
   every exception), entry-point disable path, owner read preserved. The
   legacy `additiveMigrations` boolean can no longer authorize GO — it
   masked non-additive policy removals such as the intentional
   `20260709180000` owner-only hardening.

Anything less is HOLD. `--allow-partial` refreshes a HOLD receipt but can
never mint GO — a full `release:pheno:receipt` run must confirm it.

## Safety

- Missing inputs are never treated as PASS; the preflight and runner exit
  BLOCKED (2) before any network request.
- All artifacts are redacted evidence only and live under gitignored
  `artifacts/`; sessions live under gitignored `e2e/.auth/`.
- The preflight warns if `SUPABASE_SERVICE_ROLE_KEY` is present — the live
  smoke never uses it.
- Playwright traces are disabled on real-auth runs.
